# 日記処理プロンプト

音声データの文字起こしをClaudeに渡すと、日記ファイルを自動生成・整理します。

---

## 使い方

### 日次（毎日）
音声の文字起こしをそのまま貼って依頼：

```
下記の音声文字起こしを日記として整理してください。

【文字起こし】
（ここに貼り付け）
```

### 週次（毎週末）
```
今週分の日記を読んで週次分析をしてください。

【対象ファイル】
/Users/kawairyouhei/ナレッジ/事務・運営/日記/YYYY/MM/YYYY-MM-DD.md
（7日分）
```

---

## 音声を話すときの推奨順番

毎回この順番で話すと、Claudeが構造化しやすくなります：

```
① 今日の振り返り（行動と結果）
② 売上・数字（具体的な数字を必ず言う）
③ 意思決定（今日下した判断・なぜそうしたか）
④ セールス（成約・失注・商談）
⑤ 発信（コンテンツ・SNS・アイデア）
⑥ 人間関係（サロン生・パートナー）
⑦ エネルギーログ（満たされた・失われた・得られた）
⑧ 体調
⑨ 今日の学び・気づき
⑩ 明日の戦略・最重要タスク
```

**NG例：** ダラダラ話す・順番バラバラ・感情だけ  
**OK例：** 見出しごとに区切る・数字を必ず言う・結論→理由の順

## Claudeへの処理指示（内部用）

### 保存先
- 日記本体: `/Users/kawairyouhei/ナレッジ/事務・運営/日記/YYYY/MM/YYYY-MM-DD.md`
- テンプレート: `/Users/kawairyouhei/ナレッジ/事務・運営/日記/_テンプレート.md`
- 週次分析: `/Users/kawairyouhei/ナレッジ/事務・運営/日記/_週次分析テンプレート.md`

### 処理ルール

1. **テンプレートに沿って11項目で構造化する**
   - 話し言葉はそのまま活かし、意味が変わらない程度に整える
   - 数字は必ずテーブルに整理する

2. **タグを自動付与する（冒頭に記載）**
   - 内容に応じて以下から選択・追加：
   - `#セールス` `#失注` `#成約` `#高単価` `#コンテンツ` `#サロン生`
   - `#マッチングアプリ` `#体調不良` `#学び` `#意思決定` `#発信`

3. **コンテンツアイデアが含まれる場合**
   - 日記の「5. 発信」に記載する
   - 具体的なら `/Users/kawairyouhei/ナレッジ/コンテンツ/` への転記を提案する

4. **サロン生の固有名詞・個人情報が含まれる場合**
   - 日記には記載してよい
   - 顧客管理ファイルへの転記が必要なら確認を取ってから行う

5. **外部記事・URLへの言及が含まれる場合**
   - `/Users/kawairyouhei/ナレッジ/コンテンツ/外部参考資料/` への追加を提案する

6. **ビジネス数字が含まれる場合**
   - テーブルに整理して記録する
   - 月末には `/Users/kawairyouhei/ナレッジ/経営ダッシュボード/` への集計を提案する

### 週次分析のルール

7日分の日記を読んで、以下を分析・出力する：
- 今週の売上合計と要因
- 成約パターン・共通点
- 失注・うまくいかなかった共通点
- エネルギーの増減パターン
- 改善すべき行動TOP3
- 来週の戦略・注力ポイント

結果は `/Users/kawairyouhei/ナレッジ/事務・運営/日記/YYYY_W(週番号)_週次分析.md` に保存する。
