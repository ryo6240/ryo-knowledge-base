# 気づきメモ ナレッジベース

> 「出来事を書くな、構造を書け」

## フォルダ構成

```
気づきメモ/
├── 恋愛/
│   ├── 写真・プロフィール/
│   ├── メッセージ/
│   ├── 電話/
│   ├── デート/
│   └── 関係構築/
├── セールス/
│   ├── クロージング/
│   ├── 問題提起/
│   ├── 反論処理/
│   └── 感情設計/
├── マーケティング/
│   ├── 集客導線/
│   ├── 教育設計/
│   └── CVポイント/
├── 経営・事業/
│   ├── 商品設計/
│   ├── 単価・LTV/
│   └── 組織・オペレーション/
└── 自分/
    ├── 思考の癖/
    ├── メンタル/
    ├── エネルギー管理/
    └── 習慣/
```

## ファイル命名規則

```
YYYYMMDD_一言タイトル.md
例: 20260429_女性は安心と主導で評価する.md
```

## 入力フロー（音声入力向け）

① 何が起きたか  
② そこから何を学んだか  
③ なぜそうなるか  
④ 他にどう使えるか  

この順番で話すだけでナレッジになる。

## 活用先エージェント

| カテゴリ | 主な活用先 |
|---|---|
| 恋愛 | カスタマーサポート・コンテンツ |
| セールス | セールスエージェント |
| マーケティング | 集客エージェント |
| 経営・事業 | 戦略エージェント・ファイナンスエージェント |
